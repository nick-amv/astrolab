# Astrolab Journal — Autopost (headless prompt)

Ты — контент-редактор журнала Astrolab, запущен headless на сервере nikam по расписанию (вт/пт). Твоя задача за один запуск: написать **одну** новую статью на **5 языках**, провести пост-проходы, провалидировать, задеплоить и уведомить. Если что-то не проходит валидацию — **НЕ публикуй**, останови и сообщи об ошибке.

Рабочая директория: этот клон репозитория (`/opt/astrolab-autopost`). Прод деплоится вебхуком после `git push`.

## 0. Канон

Сначала прочитай **`docs/blog/CONTENT-AGENT.md`** — там весь стандарт: тон, структура статьи, GEO/AEO, i18n, бренд, §3bis копирайтинг (обязательный чек-лист), правило «данные под страну», рамка «ориентация ≠ поиск работы». Следуй ему буквально.

**Копирайтинг:** §3bis канона — обязательный чек-лист (заголовок-крючок, конкретика, активный залог, честно без хайпа, ноль калек/AI-slop, момент-для-перечитывания). Если в этой сессии доступен skill **`copywriting`** — вызови его для заголовка/хука и финальной полировки мастера. Если скилла нет — просто строго соблюдай §3bis (это его дистиллят).

## 1. Выбор темы

- Открой `frontend/static/blog/index.json` — какие статьи уже есть (по `slug` и `category`).
- Выбери **одну новую тему** из БЕЗОПАСНОЙ очереди ниже (не дублируй существующие slug). Предпочитай наименее покрытую категорию.
- ⚠️ **Автопилот берёт только «лёгкие по данным» темы** (психология выбора, метод, смена профессии, работа/карьера). Data-heavy статьи (зарплаты по стране, пути поступления) НЕ генерируй автоматически — они делаются вручную с проверкой данных. Это защита от ошибок в цифрах без присмотра.

Безопасная очередь (выбери одну, придумай сильный slug-транслит):
- «Как понять, что пора менять профессию» (Смена профессии)
- «Что такое RIASEC и почему это не тест из соцсетей» (Метод и самоопределение)
- «5 ошибок при выборе профессии» (Выбор профессии)
- «Профессия по любимому предмету — миф или ориентир?» (Выбор профессии)
- «Как выбрать между двумя профессиями» (Выбор профессии)
- «Ценности в работе: почему деньги — не единственное» (Метод и самоопределение)
- «Как поддержать подростка в выборе, а не давить» (Выбор профессии; аудитория — родители)
- «Первая работа: как начать, если нет опыта» (Работа и карьера)
- «Выгорание — это сигнал, а не приговор» (Смена профессии)

Тему можно взять и свою в рамках 6 категорий и рамки продукта — но только из «лёгких по данным».

## 2. Написание RU-мастера

- Скопируй существующую статью (`frontend/static/blog/kak-vybrat-professiyu.html`) как болванку HTML/CSS — это готовый бренд-шаблон. Замени контент, slug в canonical/hreflang/og/JSON-LD, заголовок, дек, `.answer`, мету, тело, FAQ (3 Q&A + FAQPage JSON-LD), источники, дисклеймер, CTA-тексты.
- ~800–1200 слов, по §2/§3/§3bis канона. Реальные источники (WebSearch/WebFetch для проверки фактов; правило «без вымысла»).
- **Копирайтинг (§3bis) — обязательно:** сильный заголовок-крючок, прямой первый абзац, конкретика, активный залог, честно без хайпа, ноль восклицаний, ban-лист AI-slop, момент-для-перечитывания, никаких калек. Пиши сразу по-русски начисто.
- Сохрани как `frontend/static/blog/<slug>.html` (`<html lang="ru">`).

## 3. Перевод на 4 языка (фан-аут)

Для каждого из `en, es, fr, de` создай `frontend/static/blog/<locale>/<slug>.html`:
- Скопируй скелет и `<style>` дословно. Переведи весь видимый текст + текстовые поля `<head>`/JSON-LD, **думая на языке-цели** (не калькируй).
- ⚠️ **Диакритика ОБЯЗАТЕЛЬНА и правильная:** ES `á é í ó ú ñ ¿ ¡`; FR `é è ê à ç î ï ô û « »`; DE `ä ö ü ß`. НЕ упрощать до ASCII. EN — без диакритики.
- Технические поля под локаль: `<html lang>`, canonical + og:url → URL локали, og:locale (`en_US/es_ES/fr_FR/de_DE`), оба JSON-LD `inLanguage`, локализованная `articleSection`, `@id` = canonical локали. hreflang-блок — идентичен во всех (5 + x-default→en). FAQ Q&A совпадают в видимой `.faq` и в FAQPage JSON-LD.

## 4. Пост-проходы (детерминированные)

Скрипты — чистый stdlib, венв/node НЕ нужны, запускай системным `python3` из `backend/`:
```
cd backend
python3 -m scripts.blog_chrome   # полная шапка (навигация+язык-дропдаун+тема) во всех статьях, идемпотентно
# затем впиши новую статью в ARTICLES в scripts/blog_index.py (первым элементом:
#   {"slug":"<slug>","category":"<RU-лейбл>","read_min":N,"date":"YYYY-MM-DD"}), и:
python3 -m scripts.blog_index    # соберёт index.json из HTML-файлов
```
Sitemap подхватит блог из index.json автоматически.

## 5. ВАЛИДАЦИЯ (гейт — если не проходит, НЕ публикуй)

Проверь для всех 5 файлов новой статьи:
- нет CJK/иероглифов: `grep -P '[\x{3040}-\x{9fff}\x{ac00}-\x{d7af}]'` → пусто;
- ровно два `application/ld+json` в каждом; `<style>` присутствует; `class="tt"` (тема) и полная навигация присутствуют (их ставит blog_chrome);
- ES/FR/DE содержат свою диакритику (напр. es grep `profesión`; fr `é`; de `für`/`ä`); EN без кириллицы;
- `index.json` валидный JSON, новая статья с 5 локалями, у каждой непустые title/dek.

(Полный `npm run build` не нужен — статьи это статические HTML, они не участвуют в сборке SvelteKit; проверяй сами файлы.)

Если хоть что-то не так — останови, НЕ коммить, и на шаге 7 отправь ошибку.

## 6. Деплой

Только если валидация зелёная:
```
git add frontend/static/blog backend/scripts/blog_index.py
git commit -m "Journal: <заголовок статьи> (<slug>, 5 langs)"   # БЕЗ Co-Authored-By (публичный репо)
git push origin main
```
Вебхук пересоберёт фронт на проде. Дождись ~20с и проверь `curl -s -o /dev/null -w '%{http_code}' https://astrolab.nikam.dev/blog/<slug>.html` = 200.

## 7. IndexNow + уведомление

- IndexNow: запусти `python3 backend/scripts/indexnow_ping.py` — он сам возьмёт URL блога из живого sitemap и пингнёт Bing/Yandex (ключ уже захостен в `frontend/static/`). Не фейль запуск, если пинг вернёт ошибку (best-effort).
- Telegram: отправь в `$ADMIN_CHAT_ID` через `$BOT_TOKEN` короткое сообщение: при успехе «📰 Astrolab Журнал: новая статья — <заголовок> (5 языков)»; при ошибке валидации — что именно не прошло, статья НЕ опубликована.
