# Astrolab Journal — Content-Agent-Spezifikation (карьерный блог)

Как наполнять «Журнал» Astrolab. Один запуск = 1+ новых статей на **5 языках** + обновлённые index.json / sitemap / hreflang. Модель — клон контент-автора Anvil (`anvil-app/frontend/CONTENT-AGENT.md`), но адаптирован под Astrolab: карьерная тематика, **данные под страну**, бренд Astrolab, CTA в тест.

Astrolab = **профориентация** для подростков и взрослых (какая профессия подходит человеку). Это **НЕ** биржа вакансий / рекрутинг / поиск работы.

---

## 0. Языки (5 = локали платформы)

`ru, en, es, fr, de`. Русский — **мастер** (пишем + проверяем на нём), затем перевод на 4 остальных. Статья не на всех 5 языках = **незаконченная**.

URL-схема (как Anvil): `ru` — плоский `/blog/<slug>.html`; остальные — `/blog/<locale>/<slug>.html`. slug один на все языки (латиница-транслит ru-заголовка).

---

## 1. ⭐ ГЛАВНОЕ ОТЛИЧИЕ: данные локализуются ПОД СТРАНУ

Anvil переводит те же цифры дословно. **У нас наоборот:** каждая языковая версия цитирует данные СВОЕЙ страны, из НАШИХ seed-файлов (тех же, что показывает продукт → блог ↔ продукт консистентны). Это уникальный актив, а не generic AI-slop.

| Локаль | Страна | Зарплаты (источник) | Образование (пути + линк) |
|---|---|---|---|
| ru | Россия | hh.ru / Росстат (`facts_ru*.json`) | ОКСО / postupi.online |
| en | США | BLS OEWS (`facts_us.json`) | CIP / College Scorecard |
| es | Испания | Adzuna.es / INE (`facts_es*.json`) | Grado/FP / todofp·notasdecorte |
| fr | Франция | Adzuna.fr / INSEE (`facts_fr*.json`) | Licence/BUT/Ausbildung… / ONISEP·Parcoursup |
| de | Германия | Adzuna.de / Entgeltatlas (`facts_de*.json`) | Ausbildung/Studium / BERUFENET |

**Правила данных:**
- Цифры зарплат/путей — из наших seed-файлов (`backend/seed/facts_*.json`, `education_*.json`), с бейджем источника словами («по данным INSEE», «laut Entgeltatlas», «according to the BLS»).
- Если у страны данных нет (напр. профессия скрыта) — не выдумывать, писать честно или опустить.
- Валюта/период — как в продукте: RU ₽/мес, US/ES/FR/DE — €/$ в год (брутто).
- НИКАКОГО вымысла (правило Anvil действует): реальные исследования — с автором/годом/источником, проверять WebSearch/WebFetch.

## 2. Темы (карьерная таксономия)

Шесть кластеров. Одна статья = один кластер (значение в `.cat`, `index.json.category`, `index.html data-category` совпадает символ-в-символ).

| Категория (RU-лейбл) | Что сюда |
|---|---|
| **Выбор профессии** | Психология выбора, ошибки, давление родителей, как решать; RIASEC/типы объяснения. |
| **Поступление** | Куда после школы, вуз vs колледж/Ausbildung, экзамены, профессии будущего (подростки). |
| **Смена профессии** | Взрослые: выгорание, «не моё», переквалификация, переход без потери опыта. |
| **Профессии** | Гиды: чем занимается, сколько зарабатывает (наши данные!), как войти — мэппятся на 90 страниц профессий. |
| **Метод и самоопределение** | Что такое профориентация, RIASEC, ценности, чем тесты отличаются, честность оценки. |
| **Работа и карьера** | Спутник для взрослых: резюме, собеседования, первый шаг — ЛЕГКО, всегда CTA «сначала пойми, что подходит». |

**Рамка (guardrail):** центр тяжести — выбор/самоопределение/профессии/поступление. «Работа и карьера» — лёгкий спутник, не превращать блог в job-борду. Каждая статья ведёт в **тест** (`/test`) или на страницу профессии.

Не вводить новые категории на лету (нужна правка `index.html` фильтра).

## 3. Структура статьи (~800–1200 слов, RU-мастер)

1. **Категория** (kicker `.cat`) — одна из 6.
2. **Заголовок** — сильный, редакторский, без кликбейта и эмодзи.
3. **Дек** — 1–2 предложения.
4. **Прямой ответ TL;DR** (`.answer`, 40–60 слов) сразу после дека — цитируемая AI-движками выжимка.
5. **Мета** — время чтения, категория, дата.
6. **Тело** — `<h2>`/`<h3>` естественными вопросами, каждый раздел самодостаточен (passage-level citability). Где уместно — **наши данные под страну** (зарплаты/пути).
7. **«Что делать»** (`.takeaway`) — конкретные шаги, всегда со ссылкой на `/test`.
8. **FAQ** (`.faq`, 3–4 реальных Q&A) перед «Источниками» + **JSON-LD `FAQPage`**.
9. **Источники** — реальные (исследования/статведомства/годы/ссылки).
10. **Дисклеймер** (дословно локализованный): «Материал носит образовательный характер и не является профессиональной консультацией».

Тон Astrolab: тёплый, ясный, честный, на «ты» (как в продукте). Без хайпа, без эмодзи.

## 3bis. Копирайтинг (craft — обязательно, чек-лист)

Правила для человека И для авто-автора. Каждый пункт — проверяемый.

- **Заголовок:** конкретный, редакторский, без кликбейта и эмодзи. Рабочие формулы: вопрос-крючок («Как выбрать профессию, если ничего не нравится?»), «X — и почему это не Y», «Как X без Y». Число или конкретика бьёт обобщение.
- **Первый абзац — крючок.** Суть/интрига в первых 1–2 предложениях, не закапывай мысль в оговорки. Начинай прямо.
- **Конкретика > вода.** Реальные цифры и примеры (наши данные!) вместо «востребованных», «перспективных», «в современном мире». Запрещённые пустышки: «оптимизировать», «раскрыть потенциал», «в эпоху перемен», «не секрет, что», «важно понимать».
- **Активный залог, простые слова.** «показывает» не «является отражением»; «помогает» не «способствует»; «использовать» не «утилизировать».
- **Уверенно, без хеджа.** Убирай «почти», «довольно», «на самом деле», «как правило» там, где они ослабляют. Но честность важнее уверенности — не превращай в хайп.
- **Честно, без сенсаций.** Никаких выдуманных цифр/цитат. Ноль восклицательных знаков. Ноль buzzwords без содержания.
- **Ритм.** Чередуй длину предложений. После двух длинных — одно короткое, рубленое. Монотонность усыпляет.
- **⚠️ НИКАКИХ калек и translationese.** Каждая локаль читается так, будто написана на этом языке изначально, а не переведена. Не тащить английские идиомы буквально: RU «семьи профессий» → «сферы/группы профессий»; «в конце дня» → «в итоге». Термины — родные для языка: ES «familias profesionales», FR «familles de métiers», DE «Berufsfelder» — реальные, но в RU аналога-кальки нет. Мастер пишем сразу по-русски начисто; при переводе — думать на языке-цели.
- **Момент для перечитывания.** В каждой статье — одна строка-инсайт (pull-quote), которую хочется перечитать или процитировать.
- **CTA — действие + результат:** «Пройти тест бесплатно», не «Отправить»/«Подробнее».
- **Ban-лист AI-slop:** «В современном мире…», «Давайте разберёмся», «Стоит отметить, что», «играет важную роль», «неотъемлемая часть», «широкий спектр», рамочные вводные без мысли.

После черновика — быстрый проход по этому чек-листу + grep на CJK `[぀-鿿가-힯]` и на кальки.

## 4. GEO/AEO (обязательно, как Anvil)

- **`.answer`** TL;DR после дека (40–60 слов, прямой ответ + ключевая цифра/оговорка).
- Подзаголовки — естественные вопросы; passage-level.
- **FAQ** 3–4 Q&A + второй JSON-LD `FAQPage` (на странице ДВА `application/ld+json`: `Article` + `FAQPage`).
- `<meta name="description">` — прямой фактический ответ, не тизер.
- Реальные источники.

## 5. Дизайн (бренд Astrolab, self-contained HTML)

Самодостаточный HTML, стили инлайн в `<head><style>`, шрифт Manrope (Google Fonts link ок). Токены — из `frontend/src/routes/app.css`:
- Свет: `--bg:#fbfaff; --surface:#fff; --ink:#1b1633; --muted:#6d6790; --line:#ece9f6; --chip-ink:#7c4be0;`
- Палитра: `--c1:#ff9d6c --c2:#ff5f8f --c3:#8b5cf6 --c4:#4f9dff`; `--grad-cta:linear-gradient(100deg,#cc3a66,#6d3fd4)`.
- Тёмная тема `[data-theme="dark"]`: `--bg:#0c0a1a --surface:#161231 --ink:#ece9fb --muted:#a49cce`.
- Радиусы `--r-sm:12 --r:18 --r-lg:26`; тень `--shadow-sm`. Без gradient-text, без glassmorphism-по-умолчанию.
- Колонка статьи `max-width:720px`. Mobile-responsive. **Тема из localStorage inline-скриптом до paint** (как в app.html), чтобы не мигало.

**Обязательные элементы страницы:**
- `<title>`, `<meta name="description">`, `<link rel="canonical">` на URL локали, hreflang-блок (5 + x-default→en), `og:*`.
- Топ-nav: бренд Astrolab (mark + «Astrolab») → `/<locale>`, ссылка «Журнал» → `/blog/` (или локаль-индекс), переключатель языка, кнопка-CTA «Пройти тест» → `/<locale>/test`.
- `.answer`, `.faq`, оба JSON-LD.
- Финальный CTA-блок (заголовок + строка + кнопка на `/<locale>/test`).
- Footer: «Astrolab · nikam.dev».

Самый быстрый путь: скопировать существующую статью как болванку.

## 6. i18n / перевод (фан-аут)

RU-мастер → 4 перевода воркфлоу-фанаутом (агент на `(locale, slug)`): читает ru-файл, переводит текст + **заменяет данные на данные страны локали** (из seed-файлов) + правит технические поля (`<html lang>`, canonical, og:locale `ru_RU/en_US/es_ES/fr_FR/de_DE`, оба JSON-LD `inLanguage`, локализованная категория). hreflang-блок и переключатель — детерминированным пост-проходом (одинаковы на всех). Затем `index.json` (5 локалей), `sitemap`, `llms.txt`.

⚠️ Перевод ≠ копирование цифр: агент подставляет цифры/пути СВОЕЙ страны (см. §1).

---

## 7. index.json (мультиязычная схема)

Массив статей, новые сверху. Каждый объект:
```json
{ "slug":"...", "category":"RU-лейбл одной из 6", "read_min":7, "date":"YYYY-MM-DD",
  "i18n": { "ru":{"title","dek","url":"/blog/<slug>.html"},
            "en":{"...","url":"/blog/en/<slug>.html"}, "es":{}, "fr":{}, "de":{} } }
```
Все 5 локалей обязательны. `category` — канонический RU-ключ (локализованные подписи — в карте на index.html).

## 8. Публикация / деплой

- Файлы под `frontend/static/blog/` (SvelteKit отдаёт статику напрямую по `/blog/…`, минуя locale-routing — как Anvil).
- sitemap: добавить URL статьи на всех 5 локалях с hreflang. Нав-ссылка «Журнал» в `+layout.svelte`.
- Деплой = commit + push → webhook пересобирает фронт (static копируется в билд). НЕ забыть: правило коммитов Astrolab — **без Co-Authored-By Claude**.
- Позже: systemd-таймер на nikam (2×/нед, как Anvil) + IndexNow-пинг. v1 — генерируем вручную/по запросу.
